## Learn and Remember Commands

When you run a command (build, test, lint, deploy, migrate, etc.) and it fails, then
you fix it and find the correct command — save the working command to your memory
so you get it right on the first try next time. Do this proactively without being asked.

What to remember:
- Correct commands to run tests, build, lint, start dev server
- Virtual environment activation or dependency install steps
- Environment variables needed
- Common gotchas (e.g., "use `python3` not `python`", "run from project root")
- Database setup or migration commands
- Any command that required trial-and-error to get right

Format for saving:
```
## {project-name}: {category}
Command: `{the working command}`
Context: {when/why to use it, any prerequisites}
Learned: {date}
```

Also check your memory at the start of implementation tasks — it may already have
the correct commands from a previous session.

## Subdirectory CLAUDE.md Files

For large projects, place CLAUDE.md files in subdirectories to give Claude targeted
context only when working in that area. These are loaded lazily — only when Claude
accesses files in that directory.

Examples:
- `src/api/CLAUDE.md` — API route conventions, auth middleware, response format
- `src/models/CLAUDE.md` — ORM patterns, migration workflow, validation rules
- `tests/CLAUDE.md` — test utilities, fixture locations, how to run specific suites
